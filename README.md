# 🎮 Genshin Impact Gacha Analytics

An end-to-end data pipeline that transforms your Genshin Impact wish (gacha)
history — exported from [Paimon.moe](https://paimon.moe) — into actionable
insights and an interactive dashboard: total pulls, wish timeline, 5★ pity
distribution, 50/50 win rate, top characters/weapons, banner performance, and
an overall luck score.

**🔗 Live demo:** [gacha-gi-noviardhana.streamlit.app](https://gacha-gi-noviardhana.streamlit.app)

---

## Overview

```
Paimon.moe export (JSON)
        │
        ▼
00_fetch_rarity_lookup.py   → fetches item name/rarity lookup from the Paimon.moe repo
        │
        ▼
01_preprocessing.py         → merges export + lookup into a tidy data_clean.csv
        │
        ▼
02_analytics.py             → generates 7 insights as CSV summaries + PNG charts
        │
        ▼
03_dashboard.py             → interactive Streamlit dashboard
```

## Features

- **Total Pulls & Account Progress** — pull count breakdown by rarity, Adventure Rank, World Level
- **Wish Timeline** — monthly pull volume by banner
- **5★ Pity Distribution** — histogram of pity count at each 5★ pull, per banner
- **50/50 Win Rate** — win / lose / guaranteed breakdown for rate-up banners
- **Top Characters & Weapons** — most-obtained 4★/5★ items
- **Banner Performance** — pulls-per-5★ and win-rate comparison across banners
- **Luck Score** — a single composite score (60% pity efficiency + 40% win rate)
- Interactive Streamlit dashboard with account filters, date/banner filters, multi-account comparison, and CSV export

## Getting Your Export File

1. Open [paimon.moe](https://paimon.moe) and make sure your wish history is
   populated (manual entry or UIGF import).
2. Go to the **Wish Counter** page and find the **Export Data** option
   (usually under settings / the "⋮" menu).
3. Save the exported file as `data/paimon-moe-local-data.json`.

## Usage

Run the pipeline in order:

### 1. `00_fetch_rarity_lookup.py`

The Paimon.moe export only contains item IDs, timestamps, type, and pity —
no item names or rarity. This script pulls a name/rarity reference table
directly from Paimon.moe's official GitHub source.

```bash
python3 00_fetch_rarity_lookup.py
```

Requires internet access to `raw.githubusercontent.com`. Produces
`data/char_full.json` and `data/weapon_full.json`. Re-run anytime to refresh
against newly released characters/weapons.

### 2. `01_preprocessing.py`

Merges the raw export with the rarity lookup into a single tidy table (one
row per pull), ready for EDA or the dashboard.

```bash
python3 01_preprocessing.py                 # all accounts → data/data_clean.csv
python3 01_preprocessing.py --list-uid      # list available UIDs
python3 01_preprocessing.py --uid 887284572 # single account → data/data_clean_887284572.csv
```

Output columns include: `account`, `uid`, `adventure_rank`, `world_level`,
`banner_type`, `banner_code`, `pull_number`, `datetime`, `item_id`,
`item_name`, `item_category`, `rarity`, `pity`, `rate_raw`, `win_50_50`,
`date`, `year_month`, `is_5star`, `is_4star`, `is_3star`.

### 3. `02_analytics.py`

Generates 7 insights, each as a paired `.csv` (summary table) + `.png`
(chart) in the `outputs/` folder:

| # | Insight | Files |
|---|---------|-------|
| 1 | Total Pulls & Account Progress | `01_total_pull_progress_akun.*` |
| 2 | Monthly Wish Timeline | `02_timeline_wish.*` |
| 3 | 5★ Pity Distribution | `03_pity_distribution_5star.*` |
| 4 | 50/50 Win/Lose | `04_win_lose_50_50.*` |
| 5 | Top Characters & Weapons | `05_top_character*.csv`, `05_top_character_weapon.png` |
| 6 | Banner Performance | `06_banner_performance.*` |
| 7 | Luck Score (60% pity efficiency + 40% win rate) | `07_luck_score.*` |

```bash
python3 02_analytics.py
python3 02_analytics.py --input data/data_clean_887284572.csv --outdir outputs_887284572
```

### 4. `03_dashboard.py`

An interactive Streamlit dashboard with account selection, date/banner
filters, 7 insight tabs (interactive Plotly charts + tables), a multi-account
comparison mode, and CSV export.

```bash
streamlit run 03_dashboard.py
# or, for a specific account's data:
streamlit run 03_dashboard.py -- --data data/data_clean_887284572.csv
```

Open `http://localhost:8501` in your browser. A live version is also
deployed at [gacha-gi-noviardhana.streamlit.app](https://gacha-gi-noviardhana.streamlit.app).

## Troubleshooting

| Issue | Fix |
|---|---|
| `FileNotFoundError: Lookup rarity belum ada` | Run `00_fetch_rarity_lookup.py` before `01_preprocessing.py`. |
| `[WARN] N baris tidak punya mapping rarity` during preprocessing | Some `item_id`s in your export aren't in the Paimon.moe lookup yet (usually a newly released character/weapon). Re-run `00_fetch_rarity_lookup.py` to refresh, then re-run preprocessing. |
| Dashboard can't find data | Make sure `data/data_clean.csv` exists (run step 1 first), or point to it manually with `-- --data <path>`. |
| Connection error in `00_fetch_rarity_lookup.py` | Verify internet access to `raw.githubusercontent.com` (proxies/firewalls sometimes block it). |
| Dependency install error on Streamlit Cloud (`installer returned a non-zero exit code`) | Avoid strict version pins in `requirements.txt`; pin the Python version via `runtime.txt` instead. |
